# TryHackMe Learning Notes

This directory contains personal notes created while completing TryHackMe rooms.

The purpose of these notes is **not** to document room answers or provide walkthroughs. Instead, they serve as a personal knowledge base that reinforces concepts, workflows, and troubleshooting techniques encountered during hands-on labs.

---

## Objectives

- Build a long-term cybersecurity reference
- Reinforce concepts through documentation
- Record troubleshooting experiences
- Focus on understanding rather than memorization

## Documentation Style

Each note follows a consistent structure:

- Overview
- Learning Objectives
- Core Concepts
- Workflow
- Commands Worth Remembering
- Troubleshooting Notes
- Key Takeaways
- Personal Reflection

This structure emphasizes **why** a tool or technique is used rather than simply documenting commands.

---

## Philosophy

These notes are written for future reference.

Rather than reproducing lab instructions, they focus on:

- Understanding how tools work
- Building practical workflows
- Explaining concepts in my own words
- Recording mistakes and how they were solved

The goal is to create a reusable knowledge base that can be revisited when preparing for future labs, certifications, CTFs, or professional work.
