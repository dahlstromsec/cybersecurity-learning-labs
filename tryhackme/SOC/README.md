# Security Operations Center (SOC)

This directory contains my notes, practical exercises, and investigation methodologies developed while studying Security Operations Center (SOC) concepts.

The goal is to build a practical understanding of how security analysts detect, investigate, document, and respond to security incidents using industry-standard tools and workflows.

Rather than serving as walkthroughs or room solutions, these notes are intended to become a personal knowledge base that can be referenced throughout my cybersecurity journey.

---

Current topics include:

- Alert Triage
- Security Information and Event Management (SIEM)
- Endpoint Detection and Response (EDR)
- Security Orchestration, Automation and Response (SOAR)
- Threat Intelligence
- Incident Response
- Splunk
- Elastic Stack (ELK)
- Kusto Query Language (KQL)

As I continue through the SOC Level 1 learning path, additional concepts, investigations, and practical examples will be added.

The primary objectives of this section are to:

## Investigation Philosophy

Throughout these notes I focus on one core principle:

> **Investigate based on evidence, not assumptions.**

Security alerts should be validated using multiple sources of information before determining whether activity is malicious or legitimate.

A structured investigation process leads to more consistent decisions, better documentation, and improved incident response.

---

## Learning Approach

Rather than memorizing commands or specific room content, I aim to understand:

- Why an alert was generated
- What evidence supports the alert
- Which tools provide additional context
- How analysts reach investigation decisions
- How findings should be documented and communicated

This approach encourages transferable skills that can be applied across different security platforms and environments.
