# Overview

This room introduced one of the most important sources of information in defensive security: logs.

Almost every action performed on a system leaves behind some form of record. These records allow administrators and security analysts to monitor activity, troubleshoot issues, detect attacks, and investigate security incidents.

# What are Logs?

Logs are records of events generated by operating systems, applications, network devices, and security tools.

They provide visibility into what happened on a system by recording actions such as:

- User logins
- File access
- Process execution
- Network connections
- Authentication attempts
- System errors

Without logs, investigating security incidents becomes significantly more difficult.

---

# Why Logs Matter

Logs serve multiple purposes beyond cybersecurity.

They help organizations:

- Detect suspicious activity
- Investigate incidents
- Troubleshoot system issues
- Monitor system health
- Meet compliance requirements
- Audit user activity

For a SOC analyst, logs are often the primary source of evidence during an investigation.

---

# Common Log Sources

Examples of common log sources include:

- Windows Event Logs
- Linux system logs
- Web server logs
- Firewall logs
- IDS / IPS logs
- Authentication logs
- Application logs

Each log source provides a different perspective of an event.

# Core Concepts

## Timestamp

Every log entry contains a timestamp that indicates when an event occurred.

Accurate timestamps allow analysts to reconstruct the sequence of events during an investigation.

Without synchronized time across systems, correlating events becomes much more difficult.

---

## Log Entry

Each log entry typically contains information such as:

- Timestamp
- Event source
- Severity level
- Description
- User or process involved

These details provide context when investigating suspicious activity.

---

## Correlation

Correlation is the process of combining related events from multiple log sources.

For example, a firewall log, authentication log, and Windows Event Log may all describe different parts of the same attack.

Viewing these events together provides a much clearer picture than examining them individually.

---

## Retention

Organizations often retain logs for weeks, months, or even years.

Historical logs are valuable for:

- Incident investigations
- Threat hunting
- Compliance requirements
- Identifying long-term attacker activity

---

# Good Logging Practices

Effective logging includes:

- Collecting logs from multiple sources
- Synchronizing system time
- Centralizing log storage
- Protecting log integrity
- Retaining logs for future investigations

These practices improve both security monitoring and incident response.

---

# Biggest Takeaways

The biggest lesson from this room was that logs are the foundation of defensive security.

Security tools generate alerts, but those alerts are usually based on information contained within logs.

Understanding how logs are created, collected, and analyzed is essential for investigating security incidents and identifying malicious activity.
