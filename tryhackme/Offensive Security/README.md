# TryHackMe Learning Notes

This directory contains personal notes created while completing TryHackMe rooms.

The purpose of these notes is **not** to document room answers or provide walkthroughs. Instead, they serve as a personal knowledge base that reinforces concepts, workflows, and troubleshooting techniques encountered during hands-on labs.

---

# Objectives

- Build a long-term cybersecurity reference
- Reinforce concepts through documentation
- Record troubleshooting experiences
- Focus on understanding rather than memorization

# Documentation Style

Each note follows a consistent structure where applicable:

- Overview
- Learning Objectives
- Core Concepts
- Workflow / Process
- Commands or Tools Worth Remembering
- Troubleshooting Notes (when applicable)
- Biggest Takeaways
- Personal Reflection

The emphasis is on **understanding concepts and workflows**, rather than documenting commands or reproducing room solutions.

---

# Philosophy

These notes are written as a long-term reference for my cybersecurity learning journey.

Rather than reproducing lab instructions or room answers, they focus on:

- Understanding how tools and technologies work
- Building practical workflows
- Explaining concepts in my own words
- Recording mistakes, troubleshooting, and lessons learned
- Connecting concepts across multiple rooms and domains

The goal is to create a reusable knowledge base that can be revisited when preparing for future labs, certifications (such as SC-900), CTFs, or professional work in offensive and defensive security.
